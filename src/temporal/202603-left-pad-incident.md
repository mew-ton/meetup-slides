## left-pad インシデント — 2016 年 3 月 22 日

**npm から小さなパッケージが削除された**

- 削除したのはパッケージの作者本人
- 数時間で世界中の CI ビルドが壊れ始めた
- 影響: React, Babel, 数千のパッケージ

---

## 元凶は 17 行のコード

```js
function leftpad (str, len, ch) {
  str = String(str);
  var i = -1;
  if (!ch && ch !== 0) ch = ' ';
  len = len - str.length;
  while (++i < len) {
    str = ch + str;
  }
  return str;
}
```

> 文字列を左側から埋めるだけ

---

## 引き金は技術的な失敗ではなかった

- `kik` パッケージの名前を巡り, Kik 社と npm が著者の同意なく移管
- 著者 Azer Koçulu が抗議として **270 以上のパッケージを削除**
- left-pad はその一つに過ぎなかった

**一人の開発者の個人的な抗議が世界を止めた**

---

## なぜ世界が壊れたのか

```
babel-cli
  └─ babel-register
       └─ left-pad   ← ここが消えた
```

- 毎週数百万回ダウンロードされていた
- **誰も「消えるとは思っていなかった」**

## エコシステムが学んだこと

| 変化 | 内容 |
|---|---|
| npm unpublish ポリシー変更 | 72 時間後は削除不可 |
| package-lock.json の普及 | バージョンを固定し再現性を確保 |
| `String.prototype.padStart()` | ES2017 で標準化 |
| マイクロパッケージ忌避 | `is-array` レベルの関数を自前で書く風潮に |
