## React とは？

- Meta (旧 Facebook) が開発する UI ライブラリ
- コンポーネント指向と宣言的な UI 構築が特徴
- 仮想 DOM やフック (Hooks) などの概念を普及させた

---

## Next.js とは？

- Vercel が開発する React ベースのメタフレームワーク
- SSR / SSG / ISR などのレンダリング手法を標準提供
- App Router により、React Server Components を実用化

---

## React / Next.js のニュース

**CVSS 10 相当の脆弱性が公開**

- **Server Components のシリアライズ処理**に起因する重大な脆弱性
- 認証回避や任意のコード実行 (RCE) が可能になるケースが確認される
- Next.js 15.1.x / React 19.x のパッチバージョンへ**即座にアップデートが必要**

---

## 詳細解説

**脆弱性の概要**

特定の細工された入力を Server Action 等のエンドポイントに送信することで、本来アクセスできないサーバー内部のオブジェクトを参照・実行できてしまう問題。

**対策**

- 最新パッチへの更新が必須。
- 緩和策として、入力値の厳密なバリデーションが推奨されるが、根本解決にはアップデートが必要。

# 2025 年の総括

---

## テーマ1: OSS ガバナンスの転換点

- **「機能」の進化から「組織」の進化へ**
- React Foundation の設立や、Vite / Lit などのガバナンス刷新が相次ぐ
- 多くの主要 OSS が「特定企業への依存」から「持続可能な公的資産」へと移行
- メジャーバージョンアップと並び、エコシステムの健全化が進んだ 1 年でした

---

## テーマ2: HTML/CSS への回帰と Web Components

- **標準機能の進化と、広がるブラウザ格差**
- CSS の表現力は飛躍したが、最新機能のブラウザ間差異はむしろ拡大傾向にある
- **Design System における Web Components の採用増**
  - 多くの公開ライブラリが、特定 F/W に依存しない基盤として Web Components を採用し始めた
- JS の役割が「必須機能の実装」から「標準技術の補完」へと変化しつつある

<!-- _footer: "出典: [CARTA HOLDINGS TechBlog](https://techblog.cartaholdings.co.jp/entry/2025/12/03/163425)" -->

---

## テーマ3: セキュリティとガバナンス

- **サプライチェーン攻撃の日常化**
- npm パッケージへの悪意あるコード混入や、放置されたライブラリの乗っ取りが継続的な脅威に
- 企業における OSS 利用のガバナンス (SBoM, ロックファイルの管理) が必須スキルに
- 今月の React/Next.js の件も含め、「入れたら終わり」ではなく「運用し続ける」覚悟が問われた 1 年

---

## テーマ4: 基盤の刷新 (Infrastructure Renewal)

- **ネイティブコードへの回帰 (JS to Native/Go/Rust)**
- **TypeScript Native (tsgo)**: MS が "Project Corsa" を公開。v7.0 は Go 製となり、10倍近く高速な「待ち時間ゼロ」の DX が目前に
- **Rolldown**: 12月リリースの Vite 8 Beta にて、Rust 製バンドラ Rolldown がついにデフォルト採用など。Esbuild/Rollup の時代が終わり、統一ツールチェーンへ
- **Remix v3**: React ベースのフレームワークが **「脱 React」** を宣言

---

## まとめ

- 2025 年は「落ち着き」と「成熟」の年でした
- 派手な新技術よりも、標準技術 (Web Components/Web Platform) の底上げが進みました
- 2026 年は、この強固になった地盤の上で、再び新しいアプリケーションの形 (AI Native App など) が模索されるでしょう

また来年!
