## axios

2026/03/31

---

## "axios" とは

- JavaScript の HTTP クライアントライブラリ
- ブラウザ・Node.js 両対応で, Promise ベースの API が特徴
- npm の週間ダウンロード数は 5,000 万以上, 最も広く使われるライブラリの一つ

---

## axios のニュース

**npm パッケージがサプライチェーン攻撃に遭い, RAT が配布された**

- 攻撃者がソーシャルエンジニアリングでメンテナーの npm 認証情報を窃取
- 悪意ある v1.14.1 / v0.30.4 が約 3 時間 npm に存在
- インストールすると macOS・Windows・Linux に RAT (遠隔操作マルウェア) が仕込まれる

<!-- _footer: "出典: [Supply chain compromise of axios npm package](https://github.com/axios/axios/issues/10636)" -->

---

## 攻撃の流れ

1. **ソーシャルエンジニアリング** → メンテナーの PC に侵入
2. **npm 認証情報を窃取** → 公式パッケージとして悪意あるバージョンを公開
3. **`plain-crypto-js@4.2.1` を依存として注入** → インストール時に RAT をダウンロード・実行

3 時間で削除されたが, その間に `npm install` したプロジェクトは影響を受ける可能性がある

---

## 確認ポイント

| チェック項目 | 対応 |
|---|---|
| `axios@1.14.1` または `0.30.4` を使っている | 即時バージョンアップ + 認証情報のローテーション |
| ロックファイルに上記バージョンが含まれる | マシンごと感染の可能性あり → 完全リセット推奨 |

## axios

2026/04/10

---

## "axios" とは

- JavaScript の HTTP クライアントライブラリ
- ブラウザ・Node.js 両対応で, Promise ベースの API が特徴
- npm の週間ダウンロード数は 5,000 万以上, 最も広く使われるライブラリの一つ

---

## axios のニュース

**CVE-2026-40175 — CVSS 9.9 のヘッダーインジェクション脆弱性**

- HTTP ヘッダーの不適切なサニタイズにより, クラウドのメタデータ (AWS IAM など) が外部に漏洩する経路が存在
- v1.15.0 / v0.31.0 以降で修正済み
- ただし **標準的な Node.js 環境では実際の悪用は困難** との指摘もあり

<!-- _footer: "出典: [CVE-2026-40175 Detail - NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-40175)" -->

---

## 悪用シナリオ例

```js
await axios.get(url, {
  headers: { Authorization: `Bearer abc\r\nHost: 169.254.169.254` }
})
/**
 * 実際のリクエストはこうなる
 * GET / HTTP/1.1
 * Authorization: Bearer abc
 * Host: 169.254.169.254
 */
```

元の `Host` が上書きされ, リクエストが AWS メタデータエンドポイントへ誘導される
→ IAM の一時認証情報が攻撃者のサーバーへ漏洩

---

## 確認ポイント

| チェック項目 | 対応 |
|---|---|
| `axios` を使っているすべてのプロジェクト | `1.15.0` 以上 / `0.31.0` 以上へ更新を推奨 |

## Vercel

2026/04/17

---

## "Vercel" とは

- フロントエンド向けのクラウドデプロイプラットフォーム
- Next.js の開発元であり, git push だけでデプロイが完結する体験で普及
- 世界中の開発チームが本番環境として利用する商用サービス

---

## Vercel のニュース

**社内で利用する AI ツール経由で内部システムへの不正アクセス**

- 従業員が使用していた AI ツール (Context.ai) の Google Workspace 連携が侵害される
- そこを起点に Vercel 内部の一部環境変数などへアクセスされた
- ShinyHunters が関与を主張, $2M のランサムを要求しデータの売却を謳った

<!-- _footer: "出典: [Vercel April 2026 security incident](https://vercel.com/kb/bulletin/vercel-april-2026-security-incident)" -->

---

## 侵害の経路

```
Context.ai (AI ツール)
  └─ Google Workspace 連携が侵害される
       └─ 従業員の Vercel アカウントへ不正アクセス
            └─ 一部の環境変数・内部環境へのアクセス
```

- "sensitive" マークのない環境変数が対象 (機密マーク済みは影響なし)
- 侵害期間: **4/17 〜 4/19 UTC**

---

## フロントエンドへの影響

- Vercel を使っているプロジェクトは **環境変数のローテーションと監査ログの確認** を推奨
- 特に暗号資産系プロジェクトへの影響が報告されている
- 「AI ツールへのアクセス権がサプライチェーンの弱点になる」という新しいリスクを示した事例

## MDN

2026/04/06

---

## "MDN" とは

- Mozilla が運営する Web 技術のリファレンスドキュメントサイト
- HTML / CSS / JavaScript の仕様や API を網羅し, 世界中の開発者が日常的に参照
- 長年にわたり「フロントエンドの辞書」として業界標準の地位を占める

---

## MDN のニュース

**フロントエンドを React SPA から Web Components + Lit へ全面刷新**

- 旧実装 "yari": ejected CRA・絡み合った CSS など技術的負債が蓄積
- 静的コンテンツ中心の MDN に SPA は過剰 → サーバー側 HTML テンプレート + アイランドへ
- 開発サーバーの起動時間が **2 分 → 2 秒** に短縮

<!-- _footer: "出典: [Under the hood of MDN's new frontend - MDN Web Docs](https://developer.mozilla.org/en-US/blog/mdn-front-end-deep-dive/)" -->

---

## なぜ React をやめたか

- MDN は「ほぼ静的なドキュメント」 → SPA の恩恵がほとんどない
- インタラクティブな要素は一部だけ → **アイランドアーキテクチャ** が自然な選択
- Web Components は「各コンポーネントが自分の CSS/JS だけを読み込む」設計と相性が良い

---

## Web 標準を自ら実践

- MDN は Web 標準の「辞書」でありながら, 自サイトは React に依存していた
- 今回の刷新で **Web Components / Lit / Baseline** など, 自分たちが解説する技術を自ら採用
- 「ドッグフーディング」として業界へのメッセージにもなっている
