## React

2025/10/07

![bg 15%](./_assets/logos/react.png)

---

## Reactとは？

- Meta（旧Facebook）が 2013 年に公開した JavaScript ライブラリ
- コンポーネントベース設計と仮想DOMによる高速性が特徴
- Webアプリケーションのユーザーインターフェイス構築に広く使われる
- 発表当初から現在まで利用率 1 位!

---

## Reactのニュース

**「React Foundation」設立を発表**

- Meta,Microsoft,Vercel,Amazon らが参画
- React,React Native,JSXの新たな開発主体に
- ベンダー中立の立場でコミュニティをサポート

---

## React Foundation の役割

**中立的な開発主体として：**

- React の商標・開発インフラ（GitHub,CI等）の維持
- React Conf イベントの主催
- エコシステム支援プログラムの実行
- 財政的支援や助成金の拠出

**今後の予定：**
- 独立した技術的ガバナンス構造を構築
- 単一企業が過剰に代表されないような仕組みづくり

---

## なぜ今,Foundationなのか？

**これまで：** Meta が主導して開発  
**これから：** 複数企業が支える中立組織へ

- React の影響力拡大に伴う責任の分散
- エンタープライズ採用での信頼性向上
- オープンソースとしての持続可能性の担保

## Remix

2025/10/10

![bg 15%](./_assets/logos/remix.png)

---

## Remixとは？

- React Router 作者（Ryan Florence & Michael Jackson）が開発
- フルスタックWebフレームワーク
- Shopify、X.com、GitHub、ChatGPT など主要サービスで採用

---

## Remixのニュース

**Remix 3 を発表**

衝撃の方向転換：

- **React を捨てる**
- **Web標準ベースの新フレームワークへ**

Ryan Florence のコメント：
「僕らはもう、Reactがどこに向かっているのか分からなくなってきた」

---

## なぜReactから離れるのか？

**フロントエンドの複雑化**
- 「正直、全体像を把握できなくなってきた」
- エコシステムの複雑さに対する問題意識

**Web標準の進化**
- Node.jsは16歳、Reactは12-13歳
- ES Modules、Service Workers、Fetch API、Web Streamsなど
- プラットフォームが大きく進化した

---

## Remix 3 の核心アイデア

**Setup Scope（セットアップスコープ）**

```tsx
function App(this: RemixHandle) {
  // 1回だけ実行されるセットアップ
  let count = 0;
  
  // レンダー関数を返す
  return () => (
    <button on:click={() => {
      count++;
      this.update(); // 明示的に再レンダリング
    }}>
      {count}
    </button>
  );
}
```

- 状態はJavaScriptのクロージャ
- `this.update()` で明示的に再レンダリング

---

## その他の特徴

**Context API**
- 再レンダリングを引き起こさない
- Go to Definition でプロバイダーに直接ジャンプ

**Signal による非同期管理**
- `AbortController` の signal を活用
- レースコンディションをシンプルに解決

---

## 重要：Remix と React Router の関係

**React Router v6以前**
純粋なルーティングライブラリ
   
**Remix v1/v2**
React Routerをベースにしたフルスタックフレームワーク

**React Router v7**（2024年11月リリース）
Remix v2の機能を統合したフレームワーク

**Remix v3**（開発中）
完全に別物. Reactを使わない新フレームワーク

---

## 混乱しやすいポイント


❌ 「Remix v3 は Remix v2 のアップデート」  
✅ 「Remix v3 は全く新しいフレームワーク」

❌ 「React Router はただのルーティングライブラリ」  
✅ 「React Router v7 はフルスタックフレームワーク（旧 Remix からの移行先）」

## Vite

2025/10/13

![bg 15%](./_assets/logos/vite.png)

---

## Viteとは？

- 高速なビルドツール／開発サーバー
- Evan You（Vue.js作者）が開発
- ES モジュールと Rust ベースの Rolldown に依存することで, 圧倒的な速度を実現

---

## Viteのニュース

**「Vite+」を発表**

オールインワン開発ツールスイートとして登場

- `vite new` - プロジェクト/モノレポ生成
- `vite test` - Vitest統合
- `vite lint` - Oxlint（ESLintの100倍高速）
- `vite format` - Oxfmt（Prettier互換）
- `vite lib` - ライブラリバンドル
- `vite run` - モノレポタスクランナー
- `vite ui` - GUI開発ツール

---

## Vite+ の特徴

**統合されたツールチェーン**
- 全機能が最初から連携して動作
- 複雑な設定やプラグインの組み合わせが不要
- React, Vue, SvelteKit等の主要フレームワークに対応

**Rust製の超高速基盤**
- Parser, Resolver, Transformer, Bundler 全て Rust 実装

---

## モチベーション (1/3)

JavaScript／フロントエンドのツールチェーンは長年 
分断・複雑化しており, 特に大規模プロジェクト／複数チーム環境では「ツールが足かせになる」ことが増えてきた,
という指摘. 

---

## モチベーション (2/3)

各チームが異なるツール選択をしてしまい,
依存関係・セキュリティレビュー・プロジェクト間の整合性維持が困難.
複数プロジェクトの統合時にツールスタックの "マイグレーション地獄" に陥る. 

---

## モチベーション (3/3)

Vite+ はこうした状況を改善し, 「チームがプロダクト出荷に集中できる」ように,
「ツール選び・設定・調整・デバッグ」に費やす時間を減らすことを目指している.

---

## ビジネスモデルの転換

**オープンソース × 商用ライセンス**

✅ **無料対象：**
- 個人開発者
- オープンソースプロジェクト  
- 小規模ビジネス

💰 **有料対象：**
- スタートアップ（フラット年額）
- エンタープライズ（カスタム価格）

---

**2026年初頭：パブリックプレビュー**

現在アーリーアダプター募集中  
👉 viteplus.dev (https://viteplus.dev)

## Lit

2025/10/14

![bg 15%](./_assets/logos/lit.png)

---

## Litとは？

- Web Components作成のためのライブラリ
- Googleが開発,軽量・高速・標準準拠が特徴
- npm で5番目にダウンロードされるWebフレームワーク/コンポーネントライブラリ

---

## Litのニュース

**OpenJS Foundation への参加を発表**

JSConfにて正式発表
- Impact Projectとして参加
- Node.js,Electron,Webpack と同格の位置づけ
- Google単独プロジェクト → 中立的な財団へ移行

---

## OpenJS Foundation とは？

**JavaScriptエコシステムを支える非営利団体**

主要プロジェクト：
- Node.js
- Electron  
- Webpack
- Express.js
- jQuery

ミッション：オープンでアクセス可能なWebを実現

---

## 何が変わるのか？

**オープンガバナンスへの移行**

✅ **資産の移管**
- コード, ドキュメント, Webサイト, ブランド
- すべて OpenJS Foundation へ譲渡

✅ **Technical Steering Committee（TSC）設立**
- Google, Adobe, Reddit, 独立開発者で構成
- 企業の枠を超えた技術的意思決定

---

## なぜこの移行が重要？

**貢献者の心理的障壁を下げる**

- 「Google単独プロジェクト」への貢献をためらう人も
- 中立的な財団配下なら参加しやすい

**現時点での Lit 採用実績**

- Adobe Photoshop/Illustrator for Web
- Reddit,MDN,Internet Archive
- Chrome DevTools,Firefox UI,Microsoft Store
- YouTube,Google Maps,Home Assistant など

---

## Web Components の未来

最近の主要な進展：
- Scoped custom element registries（Safari実装）
- CSS module scripts（Firefox実装）
- Composed selection（Baseline 2025）
- Cross-scope ARIA references（仕様確定）

Litは今後も Web標準の推進をリード

---

## まとめ

大きな方向転換

- React ... Meta 単独 → React Foundation
- Remix ... React ベース → React を捨てた, 新フレームワーク
- Vite ... OSS → Vite+（商用化）
- Lit ... Google単独 → OpenJS Foundation
