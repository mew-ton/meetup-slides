# npm

![bg 15%](./_assets/logos/npm.png)

---

## npm とは ?

- Node.js のパッケージマネージャーとして事実上の標準
- 数百万のJavaScriptライブラリが公開されているレジストリ
- ほぼ全てのフロントエンド開発プロジェクトが依存している基盤インフラ

---

## npm のニュース

2025/09/08

chalk、debug等のパッケージ18個が一斉攻撃され、週間20億ダウンロードが影響を受けた

---

## 攻撃手口

1. **npm サポートを装った偽のフィッシングメールがライブラリ管理者に一斉送信された**
2. **メンテナーの認証情報を騙し取り**
3. **正規の権限でパッケージを乗っ取り**
4. **不正なコードを含むパッケージが公開された**
5. このライブラリに依存するプロジェクトが不正なコードを実行

---

## 結論

最も脆弱なのは人間

---

## サプライチェーンアタックとは？

- 開発や配布の過程にあるソフトウェアやライブラリなどを改ざんし、利用者に不正コードを届ける攻撃手法。
- 攻撃対象そのものではなく、信頼されたサプライヤーや依存関係を経由して侵入するのが特徴。
- 被害範囲が広がりやすく、検知・防御が難しいため、近年大きな脅威とされている。

---

## サプライチェーンアタックの例

```plaintext
今作っているアプリ
├── express (依存)
     ├── chalk (依存)
          └── debug (依存) ← ここがに脆弱性が埋め込まれたら・・
```

---

![bg 100%](./_assets/202509/dependency.png)

---

## なぜサプライチェーン攻撃が効果的か？ 1

**拡散力**

- 一つのパッケージで数万〜数十万のプロジェクトに影響
- chalk は週間3億ダウンロード = ほぼ全JSプロジェクト

---

## なぜサプライチェーン攻撃が効果的か？ 2

**信頼の悪用**

- 開発者は依存パッケージを疑わずに `npm install some-package`, `npx some-package` する
- CI/CD で自動実行される

---

## なぜサプライチェーン攻撃が効果的か？ 3

**検出困難**

- 正規のパッケージ更新に見える
- 静的解析では発見しにくい

---

## どう防ぐのか？

信頼されていないライブラリを

- インストールしない / 実行しない
- 自動インストールを防ぐ

---

## インストールしない / 実行しない

```sh
# ❌ 信頼されていないライブラリをインストール
npm install some-package
```

```sh
# ❌ 信頼されていないライブラリを実行
npx some-package
```

---

## インストールを防ぐ

- `package-lock.json` `yarn.lock` をコミットして管理する
- 自動のライブラリアップデートを数日遅らせる戦略をとる
  (e.g. renovate / dependabot にて, `minimumReleaseAge: "7 days"` などを設定する)
- `npm audit` を定期的に実行して脆弱性チェック

---

## ライブラリを公開する側は

- secretlint でコミットに秘密情報が含まれてないかチェック
- npm アカウントに 2段階認証を設定する

などなど
