## Google Chrome

2026/06/02

![bg 15%](./_assets/logos/googlechrome.png)

---

## "Google Chrome" とは

- Google が開発する, デスクトップで圧倒的シェアを持つ Web ブラウザ
- レンダリングエンジンは Blink, 新しい Web 機能の実装を最も積極的に進める
- Edge など他の Chromium 系ブラウザの基盤にもなっている

---

## Google Chrome のニュース

**Chrome 149 で CSS "gap decorations" が安定版に**

- グリッド / フレックスの **gap (余白) に区切り線を引ける** CSS 新機能
- `column-rule` を拡張し, 行方向用の `row-rule` を新設
- **Chromium 系 (Chrome / Edge) が単独先行, Safari / Firefox は未対応**
- 非対応ブラウザでは線が出ないだけ → プログレッシブエンハンスメント可

<!-- _footer: "出典: [Gap decorations: Now available in Chromium (Chrome for Developers)](https://developer.chrome.com/blog/gap-decorations-stable)" -->

---

## 何が嬉しいのか

```css
.grid {
  display: grid;
  gap: 2rem;
  column-rule: 1px solid #ddd; /* 列の間に縦線 */
  row-rule: 2px solid #000;    /* 行の間に横線 */
}
```

- 従来は **区切り線のために border + 疑似要素 + 計算** が必要だった
- gap の値が変わっても線は自動で追従するので保守が楽
- テーブル風・ノート風レイアウトが数行で書ける

## npm

![bg 15%](./_assets/logos/npm.png)

---

## "npm" とは

- Node.js 標準のパッケージマネージャ / 世界最大の JS パッケージレジストリ
- `npm install` 一発で依存を解決する, フロントエンド開発の生命線
- 現在は GitHub (Microsoft) が運営している

---

## npm のニュース

**npm v12 で install スクリプトの自動実行を既定で無効化**

- `preinstall` / `install` / `postinstall` を **デフォルトで実行しない** 方針へ
- 実行を許すパッケージだけ **allowlist で明示許可** (`npm approve-scripts`)
- 近年多発する **postinstall 経由のサプライチェーン攻撃** への直接的な対策
- 2026 年 7 月リリース予定, CI の移行が必要

<!-- _footer: "出典: [Upcoming breaking changes for npm v12 (GitHub Changelog)](https://github.blog/changelog/2026-06-09-upcoming-breaking-changes-for-npm-v12/)" -->

---

## なぜ大きいのか

- これまで `npm install` は依存パッケージの **任意コードを自動実行** していた
- これが「入れた瞬間に感染する」サプライチェーン攻撃の温床だった
- v12 以降は **「入れる」と「実行する」を分離**, 既定で安全側に倒れる
- これまで手動で付けていた `--ignore-scripts` が, ついに **デフォルト** になる

- 背景: AI が生成するアプリは **既定で Vite テンプレート**, デプロイ先は Cloudflare

## polyfill.io

2026/05/22

---

## "polyfill" とは

- 新しいブラウザ機能を, 未対応の古いブラウザでも動くよう補う互換コード
- 「欠けている機能を埋める (fill in)」ためのもの
- かつては IE 対応などで広く使われた

---

## "polyfill.io" とは

- 必要な polyfill だけを判定し, CDN から自動配信していたサービス
- `<script src="https://polyfill.io/...">` の 1 行で使えるお手軽さで普及
- **2024 年にサービスの運営が買収され, 運営元が交代した**

---

## polyfill.io のインシデント

**放置された参照が原因で, 不審なログイン画面が続出**

- 5/22 (JST) にドメインが Namecheap → GoDaddy へ移管, 再び応答を開始
- 旧 polyfill.io への参照が残るページで **HTTP 401 (BASIC 認証) ダイアログ** が出現
- 東芝・無印良品・ディスクユニオン・Yahoo! JAPAN など国内大手で報告が相次ぐ

<!-- _footer: "出典: [Polyfill.ioを放置したサイトで不審なダイアログが表示されている件 (Qiita)](https://qiita.com/Robot-Inventor/items/2f32a66f087a3bf7eaa5)" -->

---

## 何が起きていたのか

- 2024 年の運営交代後, polyfill.io は危険視され, 各所で利用停止が呼びかけられた
- しかし **`<script>` 参照を消し忘れた** サイトが大量に残存していた
- ドメインが再活性化して 401 を返すと, ブラウザが認証ダイアログを表示する
- 今回の直接の実害は認証ダイアログだが, **第三者ドメインを信頼し続けている** こと自体がリスク

---

## 教訓 — 外部スクリプトを放置しない

- 使わなくなった外部スクリプトの参照は **確実に削除する**
- CDN 配信の polyfill は **self-host** に切り替えるか, 不要なら外す
- `Content-Security-Policy` で読み込み先ドメインを制限しておく
- 「入れて終わり」ではなく, 依存と参照を **棚卸しし続ける**

# 特別企画 — フロントエンドの「担い手」の歴史

フロントエンド エンジニア集会 2026 年 6 月

---

## 今回の特別企画とは

**フロントエンドを "誰が" 支えてきたのか** を俯瞰します.

---

## 黎明期 — "ライブラリ" の時代 (〜2000 年代)

- 1995 年 **JavaScript** 誕生, その後ブラウザ戦争で実装差が乱立
- 課題は「ブラウザ間の差異をどう吸収するか」だった
- 2006 **jQuery** (John Resig, 個人) が定番化 → のちに OpenJS Foundation へ

---

## フレームワーク戦国時代 (2010 年代)

- 2010 **AngularJS** (Google), 2013 **React** (Facebook / Meta)
- 2014 **Vue.js** (Evan You, 個人), 2016 **Svelte** (Rich Harris, 個人)
- 仮想 DOM / コンポーネント指向で **SPA** が主流に, 担い手は企業と個人が混在

---

## ビルドツールとメタフレームワーク (2015〜)

- ES2015 以降, **webpack / Babel** をコミュニティ + スポンサーが支える
- 2016 **Next.js** (Vercel) — フレームワークとホスティングが一体化
- 2020 **Vite** (Evan You) / **esbuild** (Figma の Evan Wallace) が高速化を牽引

---

## フレームワーク + インフラの囲い込み

- **Next.js + Vercel** はフレームワークとインフラがセット
  → React 人気の流入を **インフラ利用ごと** 抱え込む構造
- **Vite + Void (VoidZero)** はフレームワーク非依存で土台を押さえ, インフラも狙う
  → そこを **Cloudflare が買収**, React / Vercel 圏に対抗する構図に

---

## Cloudflare の 2026

**1 年で "フロントエンドの主要ピース" を立て続けに買収**

- 1 月: **Astro** (コンテンツ向けフレームワーク) を買収
- 6 月: **VoidZero** (Vite ツールチェーン + デプロイ基盤 Void) を買収
- フレームワーク → ビルド基盤 → エッジ配信まで **一気に垂直統合**

<!-- _footer: "出典: [Cloudflare Acquires Team Behind Astro (The New Stack)](https://thenewstack.io/cloudflare-acquires-team-behind-open-source-framework-astro/)" -->

---

## 所有権は一方向ではない

| 時期 | 技術 | 動き |
|---|---|---|
| 2026/02 | **React** | Meta → React Foundation (企業 → 中立な財団) |
| 2026/06 | **Vite** | Evan You → Cloudflare (個人 → 企業) |

- 同じ年に **逆向き** の動きが起きた
- 企業が手放して中立化する流れと, 個人発が企業へ集約される流れが同居

---

## まとめ — 誰が中核を握るのか

- フロントエンドの担い手は **個人 / 企業 / 財団** を行き来してきた
- **中立化 (React → 財団)** と **集約 (Astro・Vite → Cloudflare)** が同時に進む 2026 年
- OSS は維持されても, **方向性を誰が決めるのか** は静かに移り変わっていく
