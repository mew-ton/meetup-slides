## CSS gap decorations

2026/06/02

![bg 15%](./_assets/logos/css.png)

---

## "CSS gap decorations" とは

- グリッド / フレックス / マルチカラムの **gap (余白) に区切り線を引く** 新仕様
- 既存の `column-rule` を拡張し, 行方向用の `row-rule` を新設
- これまで border や疑似要素のハックが必要だった処理を CSS だけで実現

---

## CSS gap decorations のニュース

**Chrome 149 で gap decorations が安定版に**

- `column-rule` が grid / flexbox / multicol でも使えるように
- 行の区切りを描く `row-rule` が新登場
- `rule-inset` や `*-rule-visibility-items` で線の伸縮・表示位置を制御
- 非対応ブラウザでは線が出ないだけ → プログレッシブエンハンスメント可

<!-- _footer: "出典: [Gap decorations: Now available in Chromium (Chrome for Developers)](https://developer.chrome.com/blog/gap-decorations-stable)" -->

---

## 何が嬉しいのか

```css
.grid {
  display: grid;
  gap: 2rem;
  column-rule: 1px solid #ddd; /* 列の間に縦線 */
  row-rule: 2px solid #000;    /* 行の間に横線 */
}
```

- 従来は **区切り線のために border + 疑似要素 + 計算** が必要だった
- gap の値が変わっても線は自動で追従するので保守が楽
- テーブル風・ノート風レイアウトが数行で書ける

## Safari

2026/06/08

![bg 15%](./_assets/logos/safari.png)

## "Safari" とは

- Apple の WebKit エンジンを採用した Web ブラウザ
- iOS / iPadOS では長らく唯一許可されてきたブラウザエンジン
- 毎年 WWDC で, 秋に来る次期 Safari のベータと新機能が発表される

---

## Safari のニュース

**WWDC26 で Safari 27 beta を公開 (58 の新機能 / 525 の修正)**

- **Customizable Select** — `<select>` を JS なしでフルにスタイル可能, アクセシビリティも維持
- **Scroll Anchoring** — ビューポート上にコンテンツが挿入されてもガタつかない
- `stretch` キーワード, `img sizes=auto`, Grid Lanes, HTML `<model>` 要素 など

<!-- _footer: "出典: [News from WWDC26: WebKit in Safari 27 beta (WebKit)](https://webkit.org/blog/17967/news-from-wwdc26-webkit-in-safari-27-beta/)" -->

---

## Customizable Select の意味

- これまで `<select>` の見た目は **OS / ブラウザ任せ** で, 凝った UI を作るには
  JS で独自実装するしかなかった (= アクセシビリティを犠牲にしがち)
- ネイティブ要素のまま自由にスタイルできる → **車輪の再発明が要らなくなる**
- Chrome が先行実装していた機能に Safari も追従 → **各ブラウザが揃ってきた**

## Red Hat

2026/06/01

![bg 15%](./_assets/logos/npm.png)

---

## "Red Hat" とは

- エンタープライズ Linux で知られる Red Hat (IBM 傘下) が公開する npm パッケージ群
- `@redhat-cloud-services` スコープで, 管理コンソール向けの
  **フロントエンドコンポーネント / API クライアント / 開発ツール** を配布
- React ベースのクラウド管理画面の土台として使われている

---

## Red Hat のニュース

**`@redhat-cloud-services` の 32 パッケージが侵害される**

- 6/1 に 32 パッケージ / 96 バージョンへ悪意あるコードが混入 (週 11.7 万 DL 規模)
- `preinstall` スクリプトで認証情報窃取マルウェア **"Miasma"** が自動実行
- GitHub Actions の **OIDC トークンを悪用** して正規スコープから publish (CI 侵害)

<!-- _footer: "出典: [Miasma: Supply Chain Attack Targeting RedHat npm Packages (Wiz)](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages) / [RHSB-2026-006 (Red Hat)](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006)" -->

---

## 先月の続き — Mini Shai-Hulud の新亜種

- "Miasma: The Spreading Blight" は **5 月の Mini Shai-Hulud と同系統** のワーム
- 手口も同じ: **OIDC トークンの窃取** で信頼された namespace に成りすまし
- 個人の認証情報ではなく **CI の信頼関係** を突くため, 多要素認証では防げない
- 「信頼された大手の namespace」でも安全とは限らない, という事例が続いている

---

## 我々はどう守るか (再掲)

- 依存追加・更新時は **新しいバージョンを即取り込まない** (数日寝かせる)
- `npm install` の **postinstall / preinstall を無効化** する選択肢 (`--ignore-scripts`)
- 6/1 以降に該当バージョンを入れたら, **CI シークレット / npm トークンを即ローテーション**
- GitHub Actions の OIDC 権限・公開フローを最小権限で見直す
